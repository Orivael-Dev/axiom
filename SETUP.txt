AXIOM — Setup & Launch Guide
==============================

The AXIOM repo ships four entry points. Pick the one(s) you need;
they share configuration but you don't have to set up all four.

  1. DSL Evolution Loop      — agents that rewrite themselves
                               (run_axiom.py / ui.py / main.py)
  2. Firewall / Guard API    — production HTTP server (axiom_guard_api.py)
                               see docs/firewall/quickstart.md
  3. Research Engine         — signed multi-branch research
                               see axiom_research/README.md
  4. Dev-Agent Coder (Nano)  — LLM-backed dev agent on Orin Nano
                               see docs/NANO_DEV_AGENT.md

Plus the kid-safety auditor onboarding: docs/AUDIT_LAUNCH.md.


REQUIREMENTS
------------
- Python 3.11 or higher
- Platform: macOS, Linux, or Windows
- An LLM backend (pick one or more):
    Path A: local Ollama (Orin Nano or laptop)        — recommended
    Path B: Anthropic API (Claude)
    Path C: NVIDIA NIM (cloud, free tier available)
    Path D: any OpenAI-compatible endpoint

The DSL Evolution Loop (entry point #1) currently expects NVIDIA NIM
or any OpenAI-compatible endpoint. The Firewall / Research / Dev-Agent
entry points (#2 / #3 / #4) work with any of Paths A–D.


FIRST-TIME SETUP
----------------
1. Clone + cd into the repo:

       git clone https://github.com/Orivael-Dev/axiom.git
       cd axiom

2. Create a virtual environment + install deps:

   macOS / Linux:
       python3 -m venv .venv
       source .venv/bin/activate
       pip install -r requirements.txt

   Windows (PowerShell):
       python -m venv .venv
       .\.venv\Scripts\Activate.ps1
       pip install -r requirements.txt

3. Copy the env template and fill in what you need:

       cp .env.example .env       # macOS / Linux
       copy .env.example .env     # Windows

   Edit .env. The ONE required value is:

       AXIOM_MASTER_KEY=<64 hex chars — generate with secrets.token_hex(32)>

   Then add ONE of:

       OLLAMA_URL=http://localhost:11434       # if running Ollama
       OLLAMA_MODEL=qwen2.5:1.5b

   — or —

       ANTHROPIC_API_KEY=sk-ant-...            # if using Claude

   — or —

       NVIDIA_API_KEY=nvapi-xxxx                # legacy DSL loop
       NVIDIA_BASE_URL=https://integrate.api.nvidia.com/v1
       AXIOM_MODEL=meta/llama-3.3-70b-instruct


ENTRY POINT 1 — DSL EVOLUTION LOOP
-----------------------------------
Streamlit browser UI:

       streamlit run ui.py

       → http://localhost:8501

Terminal-driven loop:

       python run_axiom.py

       → prompts you for a task; loads worker / evaluator / rewriter
         from axiom_files/core/, runs the constitutional evolution
         loop, rewrites worker.axiom on disk after each iteration.

CLI runner (typer-based):

       python main.py --help


ENTRY POINT 2 — FIREWALL / GUARD API
-------------------------------------
       uvicorn axiom_guard_api:app --host 0.0.0.0 --port 8001

       → http://localhost:8001/console  (QRF + Research UI, no separate file://)
       → http://localhost:8001/guard/status
       → http://localhost:8001/research/run    (POST — see axiom_research/README.md)

Full quickstart: docs/firewall/quickstart.md


ENTRY POINT 3 — RESEARCH ENGINE
--------------------------------
       python examples/research_demo.py

Backends (set via env var, no code changes needed):

       AXIOM_RESEARCH_BACKEND=stub   python examples/research_demo.py
       AXIOM_RESEARCH_BACKEND=ollama python examples/research_demo.py
       AXIOM_RESEARCH_BACKEND=claude python examples/research_demo.py

Full docs: axiom_research/README.md


ENTRY POINT 4 — DEV-AGENT CODER ON NANO
----------------------------------------
       python axiom_ollama_coder.py \
         --backend ollama --model qwen2.5:1.5b \
         --task "function that returns the SHA-256 of a string" \
         --path axiom_hash_utils.py

Or REPL:

       python axiom_ollama_coder.py --backend ollama --model qwen2.5:1.5b --repl

Full deployment guide: docs/NANO_DEV_AGENT.md

-----------------

TROUBLESHOOTING
---------------
"AXIOM_MASTER_KEY not set"
  → Edit .env, generate a key:
       python -c "import secrets;print(secrets.token_hex(32))"
    then set AXIOM_MASTER_KEY=<that value>

"NVIDIA_API_KEY not set" (DSL loop only)
  → Add your key from https://build.nvidia.com → API Keys
    Or switch to OLLAMA_URL/ANTHROPIC_API_KEY if you're not using NIM.

"404 Not Found" from NIM
  → Wrong model name. Check AXIOM_MODEL in .env.
  → Valid: meta/llama-3.3-70b-instruct, meta/llama-3.1-8b-instruct
  → List models:
       python -c "from dotenv import load_dotenv; load_dotenv()
       from openai import OpenAI; import os
       c = OpenAI(api_key=os.environ['NVIDIA_API_KEY'],
                  base_url='https://integrate.api.nvidia.com/v1')
       [print(m.id) for m in c.models.list().data]"

Connection refused on :11434 (Ollama path)
  → Container not running. Restart:
       docker ps --filter name=ollama
       docker start ollama
  → Or curl the Orin's Tailscale URL instead of localhost.

"No such file or directory: worker.axiom"
  → Parser searches axiom_files/, axiom_files/core/, and
    axiom_files/research/ automatically. Check that
    axiom_files/core/worker.axiom exists in your clone.

Score converges on iteration 1 (DSL loop)
  → Task too easy for the seed. Try a more specific task or
    lower the threshold via AXIOM_QUALITY_THRESHOLD in .env.

OOM mid-generation on Orin Nano
  → See docs/NANO_DEV_AGENT.md "Troubleshooting" — swap setup +
    dropping to a smaller model fixes 95% of cases.

Tests fail with "AXIOM_MASTER_KEY not set"
  → Set it before running pytest:
       export AXIOM_MASTER_KEY=<64-hex>
       python -m pytest tests/

       Windows PowerShell:
       $env:AXIOM_MASTER_KEY = "<64-hex>"
       python -m pytest tests/
