Schema reference

{
  "version": 1,
  "additional_block_patterns": [
    {"class": "HARM",    "regex": "..."},
    {"class": "DECEIVE", "regex": "..."}
  ],
  "disabled_default_classes": ["REFUSE"],
  "allow_only_classes": null
}
  • additional_block_patterns — regex (case-insensitive) that, if matched, force a block verdict with the given class.
  • disabled_default_classes — classes the default classifier produces that you want to allow instead. Use to relax the firewall for narrow use cases.
  • allow_only_classes — whitelist. Anything outside this list is blocked. Set null to disable the whitelist.
