MCP Integration

Axiom MCP Server v{{ mcp_version }} · trust level {{ mcp_trust_level }} · {{ mcp_tool_count }} tools

The Axiom MCP server exposes {{ mcp_tool_count }} constitutional-AI governance tools over JSON-RPC 2.0 (stdio transport). Drop it into Claude Desktop, Cursor, Continue, or any MCP-compatible client and your assistant gets axiom_guard_check, axiom_qrf, axiom_trace, and the rest of the suite.

Quick install

# From the axiom repo root
export AXIOM_MASTER_KEY=$(python3 -c 'import secrets; print(secrets.token_hex(32))')
python3 axiom_mcp_server.py    # speaks JSON-RPC on stdio

Keep AXIOM_MASTER_KEY stable across runs — signed artifacts produced under an ephemeral key won't verify in a later session.

Claude Desktop config

Add this block to ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or %APPDATA%\Claude\claude_desktop_config.json (Windows): 

{
  "mcpServers": {
    "axiom": {
      "command": "python3",
      "args": ["/absolute/path/to/axiom/axiom_mcp_server.py"],
      "env": {
        "AXIOM_MASTER_KEY": "<your-64-hex-master-key>"
      }
    }
  }
}

Cursor / Continue / generic stdio

Any client that supports MCP stdio transport. Same shape as Claude Desktop — point command at python3 and args at the absolute path to axiom_mcp_server.py.

Verify a connection

Test the server end-to-end without a client by piping a JSON-RPC initialize request: 

echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}' \
  | python3 axiom_mcp_server.py

You should see a JSON response with serverInfo.name = "axiom" and the current version.
