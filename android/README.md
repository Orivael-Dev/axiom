# AXIOM Sovereign Phone — Android client

Native Kotlin + Jetpack Compose app that wires the AXIOM REST server (`axiom_server.py`) into a phone. First slice of **ORVL-019 AXIOM Sovereign Phone Architecture** on a real device.

## What it does

Four tabs in the bottom navigation:

| Tab | What it calls | Why |
|---|---|---|
| **Gate** | `POST /phone/outbound` · `POST /phone/inbound` | The outbound + inbound classifier surfaces. Type a message, watch the signed `OutboundDecision` or `SovereignAlert` come back. Session-scoped so the L1→L2→L3 escalation works across consecutive blocks. |
| **Hello Op** | `POST /phone/outbound` ×4 (same `session_id`) | The ORVL-019 §4 scam-call demo, replayed live on the phone. Tap **Run demo call** → four cards animate in showing T+0 INFORM → T+3 L1 DECEIVE → T+6 L2 HARM → T+8 L3 HARM, with the accent color escalating green → amber → orange → red. Speed selector: 1x (real-time, 8 s) · 4x (default) · 10x (sprint). |
| **Status** | `GET /phone/status` · `GET /cmaa/fleet` · `GET /shield/status` | Live dashboard — device fingerprint, ANF call count, fleet trust levels, OS-shield daemon state. |
| **Settings** | `POST /phone/outbound` (health check) | Server URL · optional bearer token · connectivity probe. Persisted via DataStore. |

## Package as a single zip for Android Studio

The easiest path to a real APK is to import this directory into Android Studio. To hand the project to a colleague — or to your own desktop with Android Studio installed — zip the `android/` directory once and import the zip:

```bash
# From the repo root
zip -r axiom-sovereign-phone.zip android/ \
    -x 'android/build/*' 'android/app/build/*' 'android/.gradle/*' \
       'android/local.properties' 'android/gradle/wrapper/gradle-wrapper.jar'

# → axiom-sovereign-phone.zip (about 30 KB of source, no binaries)
```

In Android Studio: **File → New → Import Project** → pick the unzipped directory. Android Studio bootstraps the Gradle wrapper jar, downloads dependencies, and the project is ready to build → Run.

## What you need to build it from the command line

The app source ships in this repo; the build does not. To produce an APK directly:

```bash
# 1. Install JDK 17 and Android Studio (or just the Android command-line tools + SDK)
#    Download: https://developer.android.com/studio
#
#    OR on a headless box:
#       sudo apt install openjdk-17-jdk
#       Then download cmdline-tools from developer.android.com/studio#command-tools

# 2. Set ANDROID_HOME (or ANDROID_SDK_ROOT)
export ANDROID_HOME=/path/to/Android/sdk
export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools

# 3. Install the SDK platforms once
sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0"

# 4. From this directory — first build generates the Gradle wrapper jar
cd android
gradle wrapper           # one-time bootstrap (or open in Android Studio, which does this)
./gradlew assembleDebug
# → app/build/outputs/apk/debug/app-debug.apk
```

## Running it against the AXIOM Python stack

```bash
# 1. On your dev machine, start the AXIOM REST server bound to loopback
export AXIOM_MASTER_KEY=$(python3 -c "import secrets; print(secrets.token_hex(32))")
python3 axiom_server.py     # listens on 127.0.0.1:8000

# 2. Launch an Android emulator
#    Important: the emulator reaches the host's loopback as 10.0.2.2,
#    so the app defaults its base URL to http://10.0.2.2:8000
emulator -avd Pixel_7_API_34

# 3. Install + launch
./gradlew installDebug
adb shell am start -n dev.orivael.axiom/.MainActivity
```

## Default settings

| Setting | Default | Why |
|---|---|---|
| Server URL | `http://10.0.2.2:8000` | Android emulator loopback to host |
| Bearer token | empty | Server starts unauthenticated when `AXIOM_API_TOKEN` is unset (loopback only) |
| Session ID | autogenerated UUID per launch | Lets the L1→L2→L3 trajectory work across turns in the Gate tab |

For a real device (not emulator), change the URL to your machine's LAN IP and ensure the server is started with `AXIOM_API_TOKEN=<token>` + bound to a LAN-reachable interface.

## What's intentionally missing in this slice

- **HMAC client-side verification of server-signed payloads.** Server signs every response; this slice trusts the server. Future slice: load `AXIOM_MASTER_KEY` into Android Keystore, recompute every decision's signature locally.
- **CallScreeningService — the Hello Operator product.** Bigger lift, separate slice. The plumbing built here (gate + audit log) will become its in-call display.
- **On-device intent classification.** Right now the phone is a thin client; classification runs on the REST server. The ORVL-018 ANF emulator could be ported via Chaquopy (embed CPython) or rewritten in Kotlin as a follow-up.
